# Winehub – 다음 할 일 정리

배포 후 개선·추가로 하면 좋은 작업들을 정리한 문서입니다.

---

## 1. 우선 적용해 두면 좋은 것

### 1-1. **블렌딩 결과 정렬**

- **현재**: 블렌딩 추천 결과(와인·와이너리)가 데이터 순서 그대로 노출됨.
- **제안**: 지역명·와인 타입·이름(가나다·ABC) 등으로 정렬 옵션 추가. 예: "지역순" / "이름순" / "타입순" 토글 또는 드롭다운.

### 1-2. **지구본 색/크기**

- **현재**: 지구본 마커 색 `#a8323e`, 고정 높이 420px.
- **제안**:
  - 마커 색을 사이트 악센트(`--color-accent`)와 통일하거나, 지역별 색 구분.
  - 반응형: 모바일에서 지구본 높이 줄이기(예: 280px).
  - 지구본 텍스처/배경색을 다크 테마에 맞게 조정.

### 1-3. **번역 문구 다듬기**

- **현재**: 일부 문장이 어색하거나 직역 느낌.
- **제안**:
  - 한국어: "~해 드려요" 등 톤 통일.
  - 영어: 문장 길이·호칭 통일.
  - 프/이/일/중: 네이티브 감수 또는 전문 번역으로 교정.
  - UI 라벨은 짧고 일관되게 (예: "다시 고르기" vs "선택 초기화").

### 1-4. **언어 전환 시 “해당 언어만” 보이게**

- **현재**: UI는 선택 언어로 바뀌지만, 품종/와이너리/와인 이름 등은 `nameKo` / `nameEn`만 있어서 한국어·영어만 전환됨.
- **적용한 것**: 한국어 선택 시 한글 이름, 그 외 언어(영/프/이/일/중) 선택 시 영문 이름으로 통일해, 한 페이지 안에 한/영이 섞이지 않도록 함.
- **추가로 하면 좋은 것**:
  - 품종·와이너리·와인 데이터에 `nameFr`, `nameIt`, `nameJa`, `nameZh` 등을 넣고, 언어별로 해당 필드 우선 사용하면 “완전히 해당 언어만” 가능.
  - 우선은 UI 문구만 6개 언어로 정리하고, 데이터는 ko / en만 써도 됨.

## 2. 사이트 제목 (브랜딩)

- **적용**: 사이트명을 **Winehub**으로 통일해 두었음. (로고/네비: "Winehub", 한국어 "와인허브" 등)
- **후보** (Pornhub 느낌의 짧고 기억하기 쉬운 이름):
  - **Winehub** ✓ – 와인 + Hub, 가장 직관적. 추천.
  - **VineHub** – 포도덩굴(Vine) + Hub.
  - **CorkHub** – 코르크 + Hub.
  - **GrapeHub** – 포도 + Hub.
  - **WineHole** – 약간 유머/세련된 느낌 (hole = 구멍/바 같은 이미지).
  - **BottleHub** – 병 중심 이미지.

원하면 로고/파비콘/OG 이미지에 "Winehub" 타입face·컬러 가이드만 정해 두고 쓰면 됨.

## 3. 그다음에 하면 좋은 것

### 3-1. **SEO·메타**

- `index.html` 제목/설명을 Winehub 기준으로 수정.
  - 예: `<title>Winehub – 품종·와이너리·추천</title>`, `meta name="description"` 한/영 버전.
- OG(오픈그래프) 태그 추가해 SNS 공유 시 제목·설명·썸네일 나오게.

### 3-2. **접근성**

- 포커스·키보드 이동, 스크린 리더용 `aria-label` 보강.
- 지도/지구본에 "지도 보기" / "지구본 보기" 등 설명 텍스트.

### 3-3. **성능**

- 지구본(globe.gl)은 필요할 때만 로드(동적 import) – 이미 적용됨.
- 이미지 `loading="lazy"` 또는 저해상도 플레이스홀더.

### 3-4. **데이터**

- 와이너리 확장 데이터에 실제 와이너리 이름·링크를 단계적으로 채우기.
  - 현재는 "○○ 와이너리 1, 2, …" 형태 가상 데이터.
